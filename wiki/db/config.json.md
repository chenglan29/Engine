# 服务器配置文件（JSON）文档

## 文件概述

文件对应服务器核心配置文件（JSON格式），该文件用于存储服务器运行所需的各类基础配置参数，涵盖服务器端口、日志配置、Token设置、文件大小限制、WebSocket端口等核心信息，是服务器正常启动和运行的重要依据。配置参数可根据实际部署需求灵活调整

配置文件默认路径：./db/config.json。

## 配置文件完整内容
```json
{
    "port": 80,
    "logPath": "./log/",
    "logMaxSize": 10485760,
    "logBackups": 200,
    "tokenLength": 1024,
    "headSize": 10485760,
    "wsPort": 81
}
```
## 配置项详细说明
|配置项名称|数据类型|取值示例|核心用途|注意事项|
|---|---|---|---|---|
|port|Number（整数）|80|指定HTTP服务器监听的端口号，服务器启动后通过该端口接收客户端HTTP请求（对应服务器入口文件中 db("config").port 调用）|1. 端口号需为0-65535之间的有效整数；2. 若端口被其他程序占用，服务器将启动失败；3. 80端口为HTTP默认端口，无需在URL中显式指定端口|
|logPath|String|"./log/"|指定服务器日志文件的存储路径，日志工具（log4.js）将日志文件写入该目录，便于后续日志查看和管理|路径采用相对路径（相对于项目根目录），也可配置绝对路径|
|logMaxSize|Number（整数）|10485760|指定单个日志文件的最大容量（单位：字节），当单个日志文件大小达到该值时，日志工具将自动分割日志，生成新的日志文件|1. 示例值10485760字节 = 10MB；2. 取值需根据日志生成频率合理设置，避免日志文件过大或过小（过大不便查看，过小导致日志分割频繁）|
|logBackups|Number（整数）|200|指定日志文件的最大备份数量，当日志文件分割数量超过该值时，将自动删除最早的日志文件，保留最新的指定数量日志|1. 取值需结合日志生成频率和磁盘空间，避免备份过多占用磁盘空间；2. 若设置为0，将不保留备份日志，仅保留当前日志文件|
|tokenLength|Number（整数）|1024|指定系统生成的用户Token（身份令牌）的长度，用于用户身份验证、权限校验（如路由接口中的token验证|长度需足够长以保证Token安全性，示例值1024位可满足普通安全需求|
|headSize|Number（整数）|10485760|指定用户头像的最大上传大小限制（单位：字节），用于限制用户上传头像的文件大小，避免超大文件占用服务器资源|示例值10485760字节 = 10MB|
|wsPort|Number（整数）|81|指定WebSocket服务器监听的端口号，用于实现服务器与客户端的即时通讯|1. 需与HTTP服务器端口（port）区分，避免端口冲突；2. 端口号需为0-65535之间的有效整数，若端口被占用，WebSocket功能将无法正常使用|

## 配置关联说明

本配置文件与服务器入口文件、各类脚本文件密切关联，核心关联逻辑如下：

- 与服务器入口文件：入口文件通过 db("config").port 读取 port 参数，用于启动HTTP服务器；WebSocket脚本（./chunk/ws.js）读取 wsPort 参数，启动WebSocket服务器。

- 与日志脚本（./chunk/log4.js）：读取 logPath、logMaxSize、logBackups 三个参数，配置日志的存储路径、单个日志大小和备份数量。

- 与Token相关路由：tokenLength 参数用于Token生成函数

- 与头像相关路由：headSize 参数用于限制用户头像上传大小，与./chunk/route/resetUserHead.js、./chunk/route/updatePersonHead.js等路由及中间件（./chunk/middleware.js）的文件上传配置联动。

## 配置修改注意事项

- 格式规范：配置文件为JSON格式，修改时需严格遵循JSON语法，避免出现语法错误（如逗号遗漏、引号不匹配、冒号使用错误），否则服务器将无法读取配置，导致启动失败。

- 端口冲突：修改 port 或 wsPort 时，需确认修改后的端口未被其他程序占用，可通过系统命令（如netstat）查看端口占用情况。

- 重启生效：所有配置修改后，需重启服务器才能生效，否则服务器仍将使用修改前的配置参数。

- 备份建议：修改配置前，建议备份原始配置文件，避免修改错误导致服务器无法正常运行，便于快速回滚。

## 常见问题排查

- 服务器启动失败：检查 port 端口是否被占用、配置文件是否存在语法错误、db工具函数是否能正常读取配置文件。

- 日志无法生成：日志工具（log4.js）是否正确读取配置参数、服务器是否有该目录的写入权限。

- WebSocket无法连接：检查 wsPort 端口是否被占用、./chunk/ws.js脚本是否正确读取配置参数。

- 头像上传失败：检查上传的文件大小是否超过限制。
