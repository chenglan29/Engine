# /getInfo 接口文档

## 接口基本信息

|项目|详细说明|
|---|---|
|接口路径|/getInfo|
|请求方法|GET|
|核心功能|根据请求头中的token，获取当前登录用户的信息，并屏蔽密码、token等敏感字段|
|数据交互格式|响应为JSON格式，请求无需请求体|
## 请求参数说明

### 请求头（Headers）

|参数名|必选|类型|说明|
|---|---|---|---|
|token|是|String|用户身份令牌，格式要求：`用户名~加密字符串`，用于识别当前登录用户，解析后获取用户名以查询对应用户信息|
### 请求体（Body）

本接口为GET请求，无需提交请求体（无请求参数）。

## 代码逻辑解析

- **用户身份解析**：通过split("~")方法分割请求头中的token字符串，获取令牌的第一部分（用户名），以此作为查询用户信息的唯一标识。

- **用户数据读取**：通过自定义db模块的db("user")方法，读取用户集合（用户数据），并根据解析出的用户名，获取该用户的完整信息，存储至data变量。

- **敏感信息屏蔽**：将用户信息中的password（密码）和token（令牌）字段设为空字符串，避免敏感信息泄露，符合数据安全规范。

- **响应返回**：设置响应状态码为200（请求成功），并返回JSON格式响应，包含success（成功标识）和infomation（处理后的用户信息，无敏感字段）。

## 响应说明

### 成功响应

```json
{
  "success": true,
  "infomation": {
    "name": "用户名",  // 示例字段，实际字段以数据库存储为准
    "level": "root/admin/member",  // 示例字段，用户权限等级
    "password": "",  // 敏感字段，已屏蔽
    "token": "",     // 敏感字段，已屏蔽
    "otherField": "其他用户信息"  // 示例字段，其他非敏感信息
  }
}
```

状态码：200（OK），表示请求成功，infomation字段返回处理后的用户信息（无敏感字段）。

## 接口缺陷
- 缺失用户存在性校验：未验证token解析出的用户名是否在数据库中存在，若用户名不存在，会导致data为undefined，后续赋值操作报错。

- 缺失异常处理：未捕获db模块读取失败等异常，接口稳定性差，异常时无错误响应。
