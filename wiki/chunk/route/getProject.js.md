# /getProject 接口文档

## 接口基本信息

|项目|详细说明|
|---|---|
|接口路径|/getProject|
|请求方法|POST|
|核心功能|根据请求体中的项目标识，获取指定项目的路径、日志信息，以及配置中的wsPort端口号|
## 请求参数说明

### 请求头（Headers）

|参数名|必选|类型|说明|
|---|---|---|---|
|token|是|String|用户身份令牌|
### 请求体（Body）

|参数名|必选|类型|说明|
|---|---|---|---|
|project|是|Number|用于查询指定项目的相关信息，需与数据库中project集合的键名完全一致|

## 代码逻辑解析

- **核心功能说明**：接口主要用于获取指定项目的关键信息（路径、日志）和系统配置中的wsPort端口号，响应状态码固定为200，返回成功标识及相关数据。
- **数据读取逻辑**：
    - 通过db("project")读取项目集合数据，获取该项目信息。
    - 通过db("config")读取系统配置数据，直接获取配置中的wsPort（WebSocket端口号）字段。

- **响应返回**：返回JSON格式响应，包含success（成功标识，固定为true）、path（对象路径）、log（项目日志）、wsPort（WebSocket端口号）四个字段。

- **未实现功能（代码注释标注）**：
    - 权限校验：未验证当前请求用户是否拥有查看该项目的权限，任何请求均可获取项目信息，存在权限泄露风险。
    - 项目存在性校验：未验证req.body.project对应的项目是否存在，若项目不存在，会导致db("project")[req.body.project]为undefined，后续获取path、log字段会报错。

## 响应说明

### 成功响应

```json
{
  "success": true,
  "path": {},  // 示例：对象路径
  "log": [],  // 示例：项目日志信息
  "wsPort": 81  // 示例：WebSocket端口号，取自config配置
}
```

状态码：200（OK），表示请求成功，返回指定项目的路径、日志及wsPort配置。
## 接口缺陷
- 缺失权限校验：未验证请求用户的权限，任何请求均可获取项目信息，存在数据安全风险（代码注释已明确标注需补充）。

- 缺失项目存在性校验：未验证请求体中project参数对应的项目是否存在，项目不存在时会导致接口报错（代码注释已明确标注需补充）。

- 缺失请求参数校验：未验证project参数是否缺失，参数缺失时接口会报错，无明确错误反馈。
