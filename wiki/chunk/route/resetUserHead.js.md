# /resetUserHead 接口文档

## 接口基本信息

|项目|详细说明|
|---|---|
|接口路径|/resetUserHead|
|请求方法|POST|
|核心功能|接收用户头像重置请求，校验操作权限（root管理员可重置所有用户，admin管理员仅可重置member用户），校验通过后用备份头像覆盖目标用户头像，返回重置结果（代码注释标注需实现用户存在性校验）|
|数据交互格式|请求、响应均为JSON格式|
## 请求参数说明

### 请求头（Headers）

|参数名|必选|类型|说明|
|---|---|---|---|
|token|是|String|操作用户的身份令牌，格式为“用户名~token字符串”，用于解析操作用户身份及校验权限|
### 请求体（Body）

|参数名|必选|类型|说明|
|---|---|---|---|
|name|是|String|目标重置用户的姓名，对应头像文件的文件名（与/newUser接口中用于生成头像的name参数一致）|

## 代码逻辑解析

- **操作用户身份解析**：通过req.headers.token获取操作用户的token，用split("~")分割token，取分割后的第一个元素（用户名），作为操作用户的唯一标识，用于查询操作用户的权限等级。
      

- **权限校验与头像重置（root管理员权限）**：
 - 查询操作用户的level字段，若为“root”（超级管理员），则拥有所有用户头像的重置权限。
 - 读取备份头像文件，同步写入./heads/目录下，以请求体中的name为文件名，覆盖目标用户的原有头像。
 - 重置成功后，返回状态码200，success设为true，无额外提示信息。

- **权限校验与头像重置（admin管理员权限）**：
 - 若操作用户的level字段为“admin”（普通管理员），则仅当目标用户（req.body.name对应用户）的level为“member”（普通成员）时，拥有头像重置权限。
 - 权限校验通过后，执行与root管理员相同的头像重置操作（用备份头像覆盖目标用户头像）。

- 重置成功后，返回状态码200，success设为true，无额外提示信息。

## 响应说明

### 成功响应

```json
{
  "success": true
}
```

状态码：200（OK），表示权限校验通过，目标用户头像已成功重置（用备份头像覆盖）。

## 接口缺陷

- 无用户存在性校验：未校验操作用户（token解析的用户名）和目标用户（name对应用户）是否存在，用户不存在会导致接口报错。

- 无异常捕获机制：未处理fs文件读写失败、token格式错误、token缺失、name缺失等异常，异常时接口报错，无用户友好提示。

- 无权限不足响应：权限不满足时接口无任何响应，前端无法判断操作结果，不符合接口设计规范。

- 无请求参数校验：未校验token和name参数是否缺失，参数缺失会导致接口报错。

- 同步写入性能问题：fs.writeFileSync为同步操作，高并发场景下阻塞线程，影响接口响应速度，可能导致请求超时。
