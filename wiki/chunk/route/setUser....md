# /setUserEmail 接口文档

## 接口基本信息

|项目|详细说明|
|---|---|
|接口路径|/setUserEmail|
|请求方法|POST|
|核心功能|接收用户邮箱设置请求，校验操作权限（root管理员可设置所有用户邮箱，admin管理员仅可设置member用户邮箱），校验通过后更新目标用户邮箱并写入数据库，返回设置结果|
|数据交互格式|请求、响应均为JSON格式|
## 请求参数说明

### 请求头（Headers）

|参数名|必选|类型|说明|
|---|---|---|---|
|token|是|String|操作用户的身份令牌，格式为“用户名~token字符串”，用于解析操作用户身份及校验权限|
### 请求体（Body）

|参数名|必选|类型|说明|
|---|---|---|---|
|name|是|String|目标设置邮箱的用户姓名（对应user集合中的键名，与/newUser接口中用于生成头像的name参数一致）|
|email|是|String|目标用户的新邮箱地址，用于更新用户的email字段，无格式校验（代码未实现）|

## 代码逻辑解析

- **用户存在性校验（代码注释标注需实现）**：代码中未实现目标用户（req.body.name对应用户）和操作用户（token解析出的用户名）的存在性校验，直接读取用户数据，若用户不存在会导致接口报错。

- **操作用户身份解析**：通过req.headers.token获取操作用户的token，用split("~")分割token，取分割后的第一个元素（用户名），作为操作用户的唯一标识，用于查询操作用户的权限等级。
      
- **权限校验与邮箱设置（root管理员权限）**：
 - 查询操作用户的level字段，若为“root”（超级管理员），则拥有所有用户邮箱的设置权限。
 - 将请求体中email字段的值，赋值给目标用户（req.body.name对应用户）的email字段，完成邮箱更新。
 - 通过fs.writeFileSync同步写入./db/user.json文件，更新用户数据库，保存邮箱修改结果。
 - 设置成功后，返回状态码200，success设为true，无额外提示信息。

- **权限校验与邮箱设置（admin管理员权限）**：
 - 若操作用户的level字段为“admin”（普通管理员），则仅当目标用户（req.body.name对应用户）的level为“member”（普通成员）时，拥有邮箱设置权限。
 - 权限校验通过后，执行与root管理员相同的邮箱更新操作（赋值email字段、同步写入数据库）。
 - 设置成功后，返回状态码200，success设为true，无额外提示信息。
## 响应说明

### 成功响应

```json
{
  "success": true
}
```

状态码：200（OK），表示权限校验通过，目标用户邮箱已成功更新并保存至数据库。

## 接口缺陷

- 无用户存在性校验：未校验操作用户（token解析的用户名）和目标用户（name对应用户）是否存在，用户不存在会导致接口报错。

- 无权限不足响应：权限不满足时接口无任何响应，前端无法判断操作结果，不符合接口设计规范。

- 无请求参数校验：未校验token、name、email参数是否缺失，参数缺失会导致接口报错。

- 无邮箱格式校验：未校验email格式合法性，可能导致无效数据存入数据库。

- 同步写入性能问题：fs.writeFileSync为同步操作，高并发场景下阻塞线程，影响接口响应速度，可能导致请求超时。
