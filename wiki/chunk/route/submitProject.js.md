# /submitProject 接口文档

## 接口基本信息

|项目|详细说明|
|---|---|
|接口路径|/submitProject|
|请求方法|POST|
|核心功能|接收项目提交请求，校验项目名称是否为空、是否重名，校验通过后创建项目数据（包含项目基本信息、创建者、路径等），同步写入项目数据库，返回提交结果（代码注释标注需实现记录代码功能）|
|数据交互格式|请求、响应均为JSON格式|
## 请求参数说明

### 请求头（Headers）

|参数名|必选|类型|说明|
|---|---|---|---|
|token|是|String|操作用户的身份令牌，格式为“用户名~token字符串”，用于解析项目创建者|
### 请求体（Body）

|参数名|必选|类型|说明|
|---|---|---|---|
|projectName|是|String|项目名称，必填，不可为空、不可与已有项目重名，作为项目的核心标识（对应项目对象的name字段）|
|projectDescription|否|String|项目描述，可选参数，无默认值，为空则存储为undefined（对应项目对象的desc字段）|

## 代码逻辑解析

- **日志打印**：通过logger.debug(req.body)打印请求体信息，用于调试，记录提交的项目相关参数。
      

- **读取项目数据**：通过db("project")读取project集合数据，获取当前已存在的所有项目列表，用于后续项目名称校验。
      

- **项目名称空校验**：
    - 判断请求体中projectName字段是否为空，若为空，返回状态码200，success设为true，提示“项目名字不能为空”（注：逻辑异常，名称为空应对应失败状态）。

- **项目名称重名校验**：
    - 定义变量f并初始化为0（用于标记是否重名，0为不重名，1为重名）。
    - 通过forEach遍历已存在的项目列表，对比每个项目的name字段与请求体中的projectName，若存在相同名称，将f设为1并终止当前循环（return仅终止当前forEach迭代，不终止整个函数）。


- **项目创建与数据写入**：
    - 若项目名称不为空且不重名，向project集合中push一个新的项目对象，包含以下固定和动态字段：
        - name：请求体中的projectName（项目名称）；
        - desc：请求体中的projectDescription（项目描述，可选）；
        - creator：解析请求头token，取split("~")分割后的第一个元素（操作用户名），作为项目创建者；
        - contributor：固定为空数组，用于存储项目贡献者；
        - date：固定为当前时间（new Date()），作为项目创建时间；
        - type：固定为"Web"，作为项目类型；
        - path：固定数组，包含一个对象{"类型":"主程序"}，用于存储项目路径信息；
        - log：固定为二维空数组，用于存储项目操作日志。
    - 通过fs.writeFileSync同步写入./db/project.json文件，更新项目数据库，保存新创建的项目信息。
## 响应说明
### 成功响应
```json
{
  "success": true
}
```
状态码：200（OK），表示项目提交成功（无额外提示）；但存在逻辑异常，该响应也会在校验失败后重复返回，需优化。

### 校验失败响应（逻辑异常）

|状态码|响应内容|错误原因|
|---|---|---|
|200|{ "success": false, "message": "项目名字不能为空" }|请求体中projectName字段为空（逻辑异常，success应为false）|
|200|{ "success": false, "message": "项目重名" }|请求的projectName与已存在的项目name重复（逻辑异常，success应为false）|

状态码：无默认异常状态码（建议优化为500），接口执行过程中出现异常时会直接报错，无对应响应信息。

## 接口缺陷

- 未实现记录代码功能：代码注释要求“需要记录代码”，但未编写项目提交相关的日志记录代码。

- 无异常捕获机制：未处理fs文件读写失败、token格式错误、token缺失、请求体参数缺失等异常，异常时接口报错，无用户友好提示。

- 无请求参数校验：未校验token是否缺失、projectName是否缺失，参数缺失会导致接口报错（如token缺失会导致split报错）。

- 同步写入性能问题：fs.writeFileSync为同步操作，高并发场景下阻塞线程，影响接口响应速度，可能导致请求超时。

- 重名校验效率低：forEach遍历所有项目，项目数量较多时，校验耗时较长，影响接口性能。
