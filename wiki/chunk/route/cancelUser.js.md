# /cancelUser 接口文档

## 接口基本信息

|项目|详细说明|
|---|---|
|接口路径|/cancelUser|
|请求方法|POST|
|核心功能|根据操作方权限，注销指定用户（删除用户数据并同步至JSON文件）|
## 请求参数说明

### 请求头（Headers）

|参数名|必选|类型|说明|
|---|---|---|---|
|token|是|String|操作方用户令牌，格式要求：`用户名~加密字符串`，用于识别操作方身份及权限|
### 请求体（Body）

|参数名|必选|类型|说明|
|---|---|---|---|
|name|是|String|待注销的用户名，需与数据库中存储的用户名完全一致|

## 代码逻辑解析

- **数据读取**：通过自定义db模块读取user集合（用户数据），存储至user变量，数据来源为本地JSON文件（./db/user.json）。

- **操作方身份解析**：通过split("~")分割请求头中的token，获取操作方用户名（token的第一部分）。

- **权限校验与操作**：
    - 超级管理员（root）：若操作方权限为root，可直接删除任意用户，删除后同步写入JSON文件，并返回200状态码与成功响应。
    - 普通管理员（admin）：仅当待注销用户权限为member（普通成员）时，可执行删除操作，操作后同步写入文件并返回成功响应。

## 权限控制规则

|操作方权限等级|可注销用户权限等级|权限限制说明|
|---|---|---|
|root（超级管理员）|root、admin、member|无限制，可注销所有用户|
|admin（普通管理员）|仅member（普通成员）|无法注销root、admin权限的用户|
|member（普通成员）|无|无注销权限，无法执行本接口操作|
## 响应说明

### 成功响应

```json
{
  "success": true
}
```
## 接口缺陷
- 缺失用户存在性校验：未验证req.body.name对应的用户是否存在，删除不存在的用户会执行无意义操作，且无响应反馈。
- 缺失异常处理：未捕获文件写入失败、db模块读取失败等异常，接口稳定性差，异常时无错误响应。
- 无权限响应缺失：当操作方无注销权限时，未返回任何响应，前端无法判断操作是否成功。
- 同步写入性能问题：fs.writeFileSync为同步操作，高并发场景下会阻塞线程，影响接口响应速度。
