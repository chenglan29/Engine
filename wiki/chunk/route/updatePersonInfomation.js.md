# /updatePersonInfomation 接口文档

## 接口基本信息

|项目|详细说明|
|---|---|
|接口路径|/updatePersonInfomation|
|请求方法|POST|
|核心功能|接收用户个人信息更新请求，解析操作用户身份，更新用户邮箱、手机号，校验原密码后更新新密码（若有），同步写入用户数据库，返回更新结果|
|数据交互格式|请求、响应均为JSON格式|
## 请求参数说明

### 请求头（Headers）

|参数名|必选|类型|说明|
|---|---|---|---|
|token|是|String|操作用户的身份令牌，格式为“用户名~token字符串”，用于解析操作用户身份，定位待更新信息的用户|
### 请求体（Body）

|参数名|必选|类型|说明|
|---|---|---|---|
|email|是|String|用户新的邮箱地址，用于更新操作用户的email字段，无格式校验|
|phone|是|String|用户新的手机号，用于更新操作用户的phone字段，无格式校验|
|password|否|String|操作用户的原密码，仅当需要更新新密码（new_password非空）时必填，用于校验密码合法性|
|new_password|否|String|用户新的密码，可选参数；若为空，则不更新密码；若非空，需提交原密码（password）进行校验|
## 代码逻辑解析

- **读取用户数据**：通过db("user")读取user集合数据，获取当前所有用户的信息，用于后续操作用户定位及信息更新。

- **解析操作用户身份**：通过req.headers.token获取操作用户的token，用split("~")分割token，取分割后的第一个元素（用户名），作为操作用户的唯一标识，定位到待更新信息的用户对象。

- **更新邮箱和手机号**：
    - 直接将请求体中email字段的值，赋值给操作用户的email字段，完成邮箱更新。

    - 直接将请求体中phone字段的值，赋值给操作用户的phone字段，完成手机号更新。

    - 该步骤无任何校验（如邮箱格式、手机号格式），无论参数是否合法，均直接更新。

- **密码更新校验与处理**：
    - 场景1：新密码为空（req.body.new_password == ""）：不执行密码更新操作，返回状态码200，success设为true，无额外提示。
    - 场景2：新密码非空，但原密码错误（req.body.password != 操作用户的password字段）：不执行密码更新操作，返回状态码200，success设为false，提示“其他资料更新完毕\n但您的原密码错误，所以更改失败”（换行显示提示信息）。
    - 场景3：新密码非空且原密码正确：将请求体中new_password字段的值，赋值给操作用户的password字段，完成密码更新，返回状态码200，success设为true，无额外提示。

- **数据同步写入**：无论密码更新是否成功、邮箱和手机号更新是否合法，最终都会通过fs.writeFileSync同步写入./db/user.json文件，更新用户数据库，保存所有已做的修改（包括邮箱、手机号的更新）。
## 响应说明

### 5.1 成功响应

```json
{
  "success": true
}
```

状态码：200（OK），包含两种场景：1. 邮箱、手机号更新成功，且不更新密码（new_password为空）；2. 邮箱、手机号更新成功，且密码更新成功（原密码正确、新密码非空）。

### 5.2 失败响应

|状态码|响应内容|错误原因|
|---|---|---|
|200|{ "success": false, "message": "其他资料更新完毕\n但您的原密码错误，所以更改失败" }|新密码非空，但提交的原密码（password）与操作用户的密码不匹配，密码更新失败（邮箱、手机号已成功更新）|
## 接口缺陷

- 无请求参数校验：未校验email、phone字段是否缺失，未校验new_password非空时password是否缺失，参数缺失会导致数据异常。

- 无数据格式校验：未校验email、手机号格式合法性，未校验新密码复杂度，可能存入无效数据。

- 无异常捕获机制：未处理fs文件读写失败、token解析错误、参数缺失等异常，异常时接口报错，无用户友好提示。

- 状态码设计不规范：失败场景（原密码错误）仍返回200状态码，不符合HTTP状态码规范（失败应对应4xx状态码）。

- 同步写入性能问题：fs.writeFileSync为同步操作，高并发场景下阻塞线程，影响接口响应速度。

- 密码校验松散：使用松散等于（!=）对比密码，可能存在类型转换异常，导致密码校验失效。
