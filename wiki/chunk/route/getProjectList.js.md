# /getProjectList 接口文档

## 接口基本信息

|项目|详细说明|
|---|---|
|接口路径|/getProjectList|
|请求方法|GET|
|核心功能|获取所有项目列表，将每个项目的path字段置为空对象（节流），返回处理后的项目列表数据|
|数据交互格式|响应为JSON格式，请求无需请求体|
## 请求参数说明

### 请求头（Headers）

|参数名|必选|类型|说明|
|---|---|---|---|
|token（可选）|否|String|用户身份令牌（建议补充token用于权限校验，格式参考「用户名~加密字符串」，当前代码未实现权限校验）|
### 请求体（Body）

本接口为GET请求，无需提交请求体（无请求参数），直接请求接口即可获取所有项目列表。

## 代码逻辑解析

- **数据读取**：通过自定义db模块的db("project")方法，读取所有项目数据，存储至temp变量（temp为所有项目的集合，键为项目标识，值为项目详情）。

- **结果节流**：通过Object.keys(temp)获取所有项目的标识，遍历每个项目，将每个项目的path（对象路径）字段置为空对象，实现结果节流。

- **响应返回**：设置响应状态码为200（请求成功），返回JSON格式响应，包含success（成功标识，固定为true）和projectList（处理后的所有项目列表）。

- **未实现功能**：
    - 权限校验：未验证当前请求用户是否拥有查看项目列表的权限，任何请求均可获取所有项目信息，存在数据安全风险。
    - 项目数据校验：未验证db("project")读取的数据是否为空（无项目时），未处理无项目场景的响应优化。

## 响应说明

### 成功响应

```json
{
  "success": true,
  "projectList": [{
        "name": "3",
        "desc": "3",
        "creator": "root",
        "contributor": [
            "root"
        ],
        "date": "2026-02-24T03:38:18.177Z",
        "type": "Web",
        "path": [],
        "log": [
            []
        ]
    }]
}
```

状态码：200（OK），表示请求成功，projectList字段返回所有项目列表，每个项目的path字段已置为空对象（屏蔽敏感信息）。
## 接口缺陷
- 缺失权限校验：未验证请求用户的权限，任何请求均可获取所有项目列表，存在数据安全风险。
- 缺失项目数据校验：未判断db("project")读取的数据是否为空，无项目时返回空对象，未给出明确提示。
