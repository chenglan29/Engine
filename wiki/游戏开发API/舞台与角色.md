# 舞台
创建的舞台储存在 game.stage 中

在 IDE 中创建舞台时，它的子节点只能为角色类，否则会出错

每一项的构建方法为
```js
res + 
`${tab(level)}<canvas ${build_attribute(item)}></canvas>
${tab(level)}<script>
${tab(level)}   game.stage["${item.ID}"] = {
${tab(level)}       HTMLDOM:document.getElementById("${item.ID}"),
${tab(level)}       draw:document.getElementById("${item.ID}").getContext("2d"),
${tab(level)}       role:${JSON.stringify(item.path)}
${tab(level)}   }
${tab(level)}   document.getElementById("${item.ID}").width = window.innerWidth;
${tab(level)}   document.getElementById("${item.ID}").height = window.innerHeight;
${tab(level)}   game.$show("${item.ID}")
${tab(level)}</script>\n`
```
- item : 该舞台在结构树中的元素
- level : 该舞台在结构树中的层级
（所以说 IDE 中的 `舞台ID`是必填项，否则可能出错）
## 渲染
`game.$show(舞台ID)`是渲染舞台的方式，它的流程是这样的
- 清除画布
- 遍历子节点并把角色渲染
# 角色
您可以在`game.stage[舞台ID].role`
通过下标索引或查找角色ID来对它进行操作，比如
```js
setInterval(()=>{
game.stage.a.role[0].几何角色LEFT++
game.$show("a")
},5)
```
